# Bourbon Buddy — Firestore Data Model

**Version:** 1.1
**Last Updated:** 2026-06-24
**Database:** Cloud Firestore (Firebase)

---

## Overview

Firestore is a document/collection NoSQL database. This document defines every collection, its document structure, field types, and the query patterns that drive the app's features. All data is structured to support the query patterns the UI needs — Firestore is query-first by design.

**Conventions used in this document:**
- `string` — plain text
- `number` — numeric (integer or float as noted)
- `boolean`
- `Timestamp` — Firestore Timestamp type (not JS Date, not string)
- `array<type>` — Firestore array field
- `map` — embedded object within a document
- `reference` — Firestore DocumentReference
- `subcollection` — a nested collection under a parent document

---

## Collection: `/users/{userId}`

One document per user. Created on first sign-in (auth trigger or client-side on first load after auth).

**Document ID:** Firebase Auth UID

```
users/{userId}
  displayName:    string
  email:          string
  avatarUrl:      string | null        // Firebase Storage download URL
  bio:            string | null
  homeRegion:     string | null        // e.g., "Charlotte, NC"
  createdAt:      Timestamp
  updatedAt:      Timestamp
```

---

## Collection: `/users/{userId}/logEntries/{entryId}`

Subcollection. One document per bourbon the user has logged.

**Document ID:** Auto-generated by Firestore

```
users/{userId}/logEntries/{entryId}
  // Core identity
  bourbonId:          string           // ID of document in /bourbons collection
  bourbonName:        string           // Denormalized for display without extra read
  distillery:         string | null
  bottler:            string | null
  category:           string           // enum: see Category Values below
  subType:            string | null    // enum: see Sub-type Values below

  // Bottle details
  ageStatement:       number | null    // years; null if NAS
  isNas:              boolean
  proof:              number | null    // e.g., 107.0
  mashBillCorn:       number | null    // 0-100
  mashBillRye:        number | null    // 0-100
  mashBillWheat:      number | null    // 0-100
  mashBillMalt:       number | null    // 0-100
  batchNumber:        string | null
  barrelNumber:       string | null
  series:             string | null

  // Purchase / experience
  entryType:          string           // enum: see Entry Type Values below
  didNotPurchase:     boolean
  purchasePrice:      number | null
  purchaseLocation:   string | null
  purchaseDate:       Timestamp | null
  bottleSizeMl:       number | null    // 50, 200, 375, 750, 1000, 1750
  bottleRemainingPct: number | null    // 100, 75, 50, 25, 0

  // Ratings and notes
  rating:             number | null    // 0.5 – 5.0, half-step increments
  wouldBuyAgain:      string | null    // enum: 'yes' | 'no' | 'maybe'
  noseNotes:          string | null
  noseTags:           array<string>    // flavor tag names
  palateTags:         array<string>
  palateNotes:        string | null
  finishTags:         array<string>
  finishNotes:        string | null
  finishLength:       string | null    // 'short' | 'medium' | 'long'
  personalNotes:      string | null
  labelPhotoUrl:      string | null    // Firebase Storage download URL

  // Computed
  valueScore:         number | null    // (rating/5)*100/purchasePrice; null if missing data

  // Metadata
  entryDate:          Timestamp        // date of the experience (user-set, defaults to today)
  createdAt:          Timestamp
  updatedAt:          Timestamp
```

**Category Values:** `bourbon` | `rye` | `wheat_whiskey` | `tennessee` | `american_other` | `scotch` | `irish` | `japanese` | `world_other`

**Sub-type Values:** `single_barrel` | `small_batch` | `blended` | `cask_strength` | `nas` | `straight` | `bottled_in_bond`

**Entry Type Values:** `drink` | `bottle_purchased` | `gift_received` | `sample_split` | `virtual_tasting`

**Query Patterns:**
- All entries for a user: `collection('users/{uid}/logEntries').orderBy('entryDate', 'desc')`
- Filter by category: add `.where('category', '==', 'bourbon')`
- Filter by rating range: `.where('rating', '>=', 4).where('rating', '<=', 5)`
- Sort by value score: `.orderBy('valueScore', 'desc')`
- Search by name: client-side filter on `bourbonName` after fetching (Firestore has no native full-text search; for MVP this is acceptable given small data set; Algolia or Typesense can be added later)

## Subcollection: `/users/{userId}/logEntries/{entryId}/pourSessions/{sessionId}`

One document per pour session from a purchased bottle. Only relevant when `entryType === 'bottle_purchased'`.

```
pourSessions/{sessionId}
  pourDate:       Timestamp
  rating:         number | null    // 0.5–5.0
  settingNotes:   string | null
  tastingNotes:   string | null
  createdAt:      Timestamp
```

**Query Pattern:** `collection('...logEntries/{entryId}/pourSessions').orderBy('pourDate', 'asc')`

## Collection: `/users/{userId}/wishlistEntries/{entryId}`

Subcollection. One document per bourbon the user wants to try.

```
wishlistEntries/{entryId}
  // Core identity
  bourbonId:              string           // /bourbons ref
  bourbonName:            string           // denormalized
  distillery:             string | null
  category:               string | null
  subType:                string | null
  msrp:                   number | null

  // Research
  externalTastingNotes:   string | null
  reviewLinks:            array<map>       // [{url: string, label: string|null}]
  personalNotes:          string | null
  discoverySource:        string | null    // e.g., "Whisky Advocate"
  discoveryUrl:           string | null

  // Priority
  priority:               string           // 'grail' | 'high' | 'normal' | 'low'
  status:                 string           // 'actively_looking' | 'casually_looking' | 'just_browsing' | 'logged'

  // Computed / cached
  bestSightingPrice:      number | null    // cached lowest non-stale price; updated when sightings change

  // Metadata
  createdAt:              Timestamp
  updatedAt:              Timestamp
```

**Query Patterns:**
- Active wishlist: `.where('status', '!=', 'logged').orderBy('priority').orderBy('bourbonName')`
- Priority sort uses custom ordering: client maps `grail=0, high=1, normal=2, low=3` or stores a numeric `priorityOrder` field

## Subcollection: `/users/{userId}/wishlistEntries/{entryId}/sightings/{sightingId}`

Price sightings tied to a specific wishlist entry.

```
sightings/{sightingId}
  storeName:      string
  price:          number
  sightingDate:   Timestamp
  city:           string | null
  state:          string | null
  notes:          string | null
  markedStaleManually: boolean    // user manually marked stale
  createdAt:      Timestamp
```

**Staleness Logic:** `isStale = markedStaleManually || (today - sightingDate > 60 days)` — computed on read, not stored.

**Query Pattern:** `.orderBy('price', 'asc').orderBy('sightingDate', 'desc')`

## Collection: `/bourbons/{bourbonId}`

Shared catalog of bourbon/whiskey products. Created by users when they log a bottle not already in the catalog. Benefits all users over time.

**Document ID:** Auto-generated

```
bourbons/{bourbonId}
  name:               string           // normalized (trimmed, consistent casing)
  nameLowercase:      string           // for case-insensitive search queries
  distillery:         string | null
  bottler:            string | null
  category:           string | null
  subType:            string | null
  ageStatement:       number | null
  isNas:              boolean
  proof:              number | null
  msrp:               number | null
  series:             string | null
  createdAt:          Timestamp
  createdByUserId:    string           // which user added this to catalog
```

**Query Patterns:**
- Name autocomplete: `.where('nameLowercase', '>=', query).where('nameLowercase', '<=', query + '\uf8ff').limit(10)`
- Distillery lookup: `.where('distillery', '==', 'Buffalo Trace')`

**Note on deduplication:** When a user types a bourbon name, the client searches `nameLowercase` for an approximate match. If found, the existing document is reused. If not, a new document is created. Exact string matching is used for MVP; fuzzy matching can be added later.

## Collection: `/newsArticles/{articleId}`

Written by the Cloud Function RSS fetcher only. Read by all authenticated users.

**Document ID:** URL-derived hash (to enable deduplication on write)

```
newsArticles/{articleId}
  sourceName:     string           // e.g., "Breaking Bourbon"
  headline:       string
  excerpt:        string | null    // 2–3 sentence summary from RSS item description
  url:            string           // original article URL (unique)
  thumbnailUrl:   string | null
  publishedAt:    Timestamp | null
  fetchedAt:      Timestamp
  categories:     array<string>   // e.g., ['release', 'award', 'general']
  keywords:       array<string>   // extracted/assigned for feed personalization matching
```

**Query Patterns:**
- Latest articles: `.orderBy('publishedAt', 'desc').limit(50)`
- Filter by category: `.where('categories', 'array-contains', 'award')`
- Keyword match: `.where('keywords', 'array-contains', 'blanton\'s')` (client-side keyword matching is more flexible for MVP)

**Cleanup:** Articles older than 90 days are deleted by a monthly scheduled Cloud Function.

## Collection: `/userNewsPreferences/{userId}`

One document per user. Stores feed customization settings.

**Document ID:** Firebase Auth UID (same as user doc)

```
userNewsPreferences/{userId}
  watchKeywords:      array<string>   // e.g., ["Blanton's", "Pappy"]
  watchDistilleries:  array<string>   // e.g., ["Buffalo Trace", "Heaven Hill"]
  activeCategories:   array<string>   // e.g., ["general", "award", "release"]
  excludeKeywords:    array<string>
  updatedAt:          Timestamp
```

**Default state on creation:** `activeCategories: ['general']`, all other arrays empty.

## Collection: `/userArticleStates/{userId}/articles/{articleId}`

Tracks per-user read/saved/dismissed state for news articles. Subcollection under a user-keyed top-level document.

**Alternative structure (simpler for security rules):** store as a subcollection under `/users/{userId}/articleStates/{articleId}` — preferred approach since all user data is already under `/users/{userId}`.

```
users/{userId}/articleStates/{articleId}
  state:        string       // 'read' | 'saved' | 'dismissed'
  updatedAt:    Timestamp
```

Only documents where the user has taken an action exist. Absence of a document = unread/unsaved.

**Query Patterns:**
- All saved articles: `.where('state', '==', 'saved').orderBy('updatedAt', 'desc')`
- All dismissed: `.where('state', '==', 'dismissed')` — fetched to exclude from feed

## Firestore Security Rules (Summary)

Full rules to be written in `firestore.rules`. Key principles:

```
// User profile and all subcollections
match /users/{userId}/{document=**} {
  allow read, write: if request.auth != null && request.auth.uid == userId;
}

// Bourbon catalog — any auth user can read or create; only creator can update
match /bourbons/{bourbonId} {
  allow read: if request.auth != null;
  allow create: if request.auth != null;
  allow update: if request.auth != null && request.auth.uid == resource.data.createdByUserId;
}

// News articles — any auth user can read; only admin/function SDK can write
match /newsArticles/{articleId} {
  allow read: if request.auth != null;
  allow write: if false; // Cloud Functions use admin SDK, bypasses rules
}

// User news preferences — own document only
match /userNewsPreferences/{userId} {
  allow read, write: if request.auth != null && request.auth.uid == userId;
}
```
